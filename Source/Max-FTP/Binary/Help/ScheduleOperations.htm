Properties of Operations

Operation Info

By setting the Operation Info you are specifying what you want to occur when the operation runs. Valid operations are Copy, Move, Delete, Folder and Rename. You can specify anything you want for the Name value; it's for internal use only. All schedule events by default are logged under the System32\LogFiles folder, and NT based system also in EventLog.

The copy operation requires that you tell Max-FTP what you want to copy. If you specify file, you'll need to enter just the filename you want to copy. If you specify folder, you'll need to enter just the last part of the folder name. You only enter the filename; or the last part of the folder the rest of the location (directory) is specified in the Source or Destination Directory URL section on the operation window.

The move operation works the save way as the copy does except it moves the files off of the source location.

Copy, move and delete operations have the option of including sub folders, this means that every folder until the root specified (apart from the root specified, the exception is delete which will remove the root specified as well) will be included in the provided operation action. Note, when using wild cards use just a star to specify folders included in the action, or use *.* to specify just all files.

Copy and move operations have the option of overwrite, which has a sub option of only newer. Select overwrite to ensure the files are overwritten, select both overwrite and only newer if you want the destination overwrote only if the source is newer then the destination.

The delete operation requires that you enter the file name you want to delete. You only enter the filename; the rest of the location (directory) is specified in the Source Directory URL section on the operation window.

The Folder operation simply creates a folder of you choice.

The Rename operation allows you to rename files or folders. You must specify the current name of the file or folder and the desired name of the file or folder. You only enter the filename; the rest of the location (directory) is specified in the Source Directory URL section on the operation window.

Source Directory URL

This is where you specify the location of the file to perform actions on. It can be local, ftp, or a network resource URL. You must specify the path to where the file resides. You do not include the filename or folder name of which you're performing action on.

Destination Directory URL

This is the same things as the Source Directory URL except it's for the Destination. This is only visible on Copy and Move operations.
