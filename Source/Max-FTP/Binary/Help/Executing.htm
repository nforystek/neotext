The Environment

Each project requires the environment to be executing, more then one environment can be loaded at once. Due to the nature of the possibilities of scripts it is possible by scenario that the project never finishes thus the nature of rise is up to the creator. All errors become displayed in the debugging window and highlight the line of code with red, with the exception of evaluated string expressions.

Executing Projects

From the project file stand point MaxIDE's installer is setup to install actions for .MPRJ files in the system. By right clicking the project you have the option to open or execute the project. Executing the project from the file causing the environment to be invisible but still runs as a normal task to the system. Any error that occurs makes the environment visible.

Debugging Projects

The debugger window allows basic interface to VB functionality but is not connected to project runtime objects, it uses a separate interpreter. You are able to use "!" plus a statement to execute it, or "?" plus an expression to evaluate it in the window. For instance Typing "?Asc("C")" and hitting enter returns the Asc value of C, likewise typing "?(4+9)" evaluates to 13.

Project Definition

By default, no project is required to have any code and the environment will still run. All MaxIDE automation and events are supplied by the templates provided and are outlined in the help documents. Aside from everything else, uniquely, Project.Finish being executed defines the ending of a projects bound, for more information see the Project Object.
