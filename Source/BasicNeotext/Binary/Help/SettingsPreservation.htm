One of the advents and great design considerations to Visual Basic 6.0 is the creation of the managed memory development environment.  Although, when applications get bigger and heavier with more complexities in unconventional “out of the box” development with Visual Basic, it can be known to crash or ghost memory faults waiting to happen.  This sometimes can jolt settings, and in my belief especially with add-ins, for instance the data connection view window in my own experience has been known from time to time set itself to open. 

 

This can especially occur with Add-In features due to the consideration of the multi-instancing and the office tool bar with respects to the Add-In connect module.  A feature in BasicNeotext has been placed under the “Window” menu in Visual Basic which is the menu item “Preserve Window Layout,” by selecting this you settings will be saved in the Visual Basic folder in a file that will automatically reapply when you open Visual Basic.  When the feature is on, the menu item will read “Release Window Layout” which relinquishes the settings form applying when the environment is opened.
