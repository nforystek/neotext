BasicNeotext places some new build menu items under the main File menu and the Run menu.  In the File menu, two menu items have been added, one is a “Remake Project Build” that allows the compile of the project selected with out having to select where to compile, this option only does so if the project has already been compiled once before, to know the location.

 

The second menu item in the File menu is the “Build Project Release” menu item which does the same thing as the “Remake Project Build” but has the option of code signing the project as well in one compile.  Code signing must be setup from the “Tools” menu by selecting the “Code Signing” menu item which will load a dialog allowing thus.

 

Under the “Run” menu, two menu items have been added, which are “Start the Executable,” and “Start with Full Compile.”  In more advanced situation especially with some debugging characteristic that is common in service practices you might find the need to be running the actual executable reference least some advent in place and/or debug queries that only occur with out the environment itself running the application.   Both of these menu options run the executable file, and the one will compile it first as well before running it.
